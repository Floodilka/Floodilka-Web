# 🔒 Шифрование в Floodilka - Шпаргалка для ответов

## 💬 Краткий ответ (для обычных пользователей)

> **"Да, все голосовые звонки в Floodilka зашифрованы. Мы используем WebRTC с AES-128 шифрованием - тот же стандарт, что и в Discord, Zoom и Google Meet. Ваши разговоры защищены от прослушивания."**

## 📊 Технический ответ (для IT-специалистов)

> **"Floodilka использует WebRTC с обязательным DTLS-SRTP шифрованием. Все медиа-потоки защищены AES-128-GCM, сигнализация через WSS (TLS 1.2+). Ключи генерируются и обмениваются через DTLS с использованием Diffie-Hellman. Уровень безопасности соответствует RFC 5763/3711."**

## 🎯 Конкретные вопросы и ответы

### "Какое у вас шифрование?"

**Ответ:**
```
WebRTC с DTLS-SRTP:
• Алгоритм: AES-128-GCM
• Протокол: DTLS 1.2 + SRTP
• Обмен ключами: Diffie-Hellman
• Тот же уровень что Discord/Zoom
```

### "Можно ли вас прослушивать?"

**Ответ:**
```
НЕТ. Трафик зашифрован между клиентами.
✅ Провайдер интернета - НЕТ (видит только зашифрованные данные)
✅ Хакеры в сети - НЕТ (AES-128 не взламывается)
⚠️ Сервер - видит только метаданные (кто, когда, но НЕ контент)
```

### "Почему не E2E как в Signal?"

**Честный ответ:**
```
Signal использует E2E (сквозное шифрование).
Floodilka использует Transport Encryption.

Разница:
• Signal: даже сервер не видит ничего
• Floodilka: сервер видит метаданные, но не контент

Зачем так:
• Проще реализация
• Лучшая производительность
• Меньше багов
• Достаточно для 99% случаев
```

### "Это безопасно для бизнеса?"

**Ответ:**
```
ДА, если:
✅ Не обсуждаете гостайну
✅ Не медицинские данные (HIPAA)
✅ Не финансовые транзакции

НЕТ, если:
❌ Военные коммуникации
❌ Критичная медицина
❌ Банковские переговоры

Для таких случаев нужно E2E шифрование.
```

## 🛡️ Как доказать безопасность?

### Способ 1: Показать в Chrome DevTools

**Инструкция для скептиков:**
```
1. Откройте chrome://webrtc-internals
2. Начните звонок в Floodilka
3. Найдите раздел "Stats"
4. Покажите:
   - dtlsState: connected ✅
   - srtpCipher: AES_CM_128_HMAC_SHA1_80 ✅
   - Это доказывает шифрование!
```

### Способ 2: Wireshark тест

**Если кто-то хочет "поймать" ваш трафик:**
```
1. Запустите Wireshark

4. Результат: все пакеты ЗАШИФРОВАНЫ

```

## ⚠️ Честные ответы на сложные вопросы

### "Можете ли ВЫ прослушивать звонки?"

**Честный ответ:**
```
ТЕХНИЧЕСКИ - нет, потому что:
• Ключи генерируются между клиентами
• Мы не имеем доступа к ключам DTLS
• Медиа-поток P2P (peer-to-peer)

НО видим метаданные:
• Кто с кем говорил
• Когда начался звонок
• Сколько длился

Для полной анонимности нужен E2E.
```

### "Что если правительство потребует данные?"

**Честный ответ:**
```
Можем предоставить только:
✅ Метаданные (кто, когда, IP-адреса)
❌ Контент разговоров (мы его не храним)

Аудио НЕ записывается.
Логи хранятся [указать срок].
```

### "А если взломают сервер?"

**Честный ответ:**
```
Если взломают сервер:
✅ Не получат аудио (оно зашифровано и не хранится)
⚠️ Могут получить метаданные
⚠️ Могут перенаправить будущие звонки

Защита:
• Регулярные обновления
• Мониторинг безопасности
• Firewall + fail2ban
```
