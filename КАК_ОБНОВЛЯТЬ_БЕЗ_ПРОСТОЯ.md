# 🚀 Как обновлять Floodilka БЕЗ простоя

## Проблема которая была

Когда вы запускали `update-graceful.sh`:
- ❌ Сервер падал на 2-5 секунд
- ❌ Пользователи видели ошибку 500
- ❌ WebSocket соединения рвались
- ❌ Голосовые звонки прерывались

**Это плохо, особенно если пользователи онлайн!**

## ✅ Решение: Zero-Downtime Deployment

Я создал систему **zero-downtime deployment** - обновление БЕЗ простоя.

### Как это работает

**Обычный способ:**
```
[Backend] → остановка → обновление → запуск
          ❌ 2-5 сек простоя
```

**Zero-Downtime:**
```
[Instance 1] ────────────────────────────> работает
[Instance 2] ────────────────────────────> работает
                     ↓
          [Новые инстансы] запускаются
                     ↓
          Healthcheck ✓
                     ↓
          [Старые инстансы] останавливаются

✅ Всегда минимум 2 инстанса онлайн = 0 сек простоя!
```

## 🎯 Что делать (пошагово)

### Шаг 1: Закоммитить новые файлы (локально)

```bash
# На вашем компьютере
cd /path/to/Floodilka

git add .
git commit -m "feat: add zero-downtime deployment"
git push origin main
```

### Шаг 2: Первое обновление на сервере (на дроплете)

```bash
# Подключиться к серверу
ssh root@your-server

# Перейти в проект
cd /var/www/floodilka

# Получить изменения
git pull origin main

# Запустить zero-downtime обновление
sudo bash deployment/update-zero-downtime.sh
```

**При первом запуске:**
- Будет простой ~2-3 секунды (ОДИН РАЗ) для настройки cluster mode
- Скрипт всё настроит автоматически
- **Все последующие обновления - БЕЗ простоя!** ✨

### Шаг 3: Проверка

```bash
# Убедиться что backend в cluster mode
sudo -u floodilka pm2 status

# Должно быть примерно так:
┌────┬──────────────────────┬─────────┬───┬────────┐
│ id │ name                 │ mode    │ ↺ │ status │
├────┼──────────────────────┼─────────┼───┼────────┤
│ 0  │ floodilka-backend    │ cluster │ 0 │ online │
│ 1  │ floodilka-backend    │ cluster │ 0 │ online │
└────┴──────────────────────┴─────────┴───┴────────┘
              ^^^^ CLUSTER MODE - два инстанса!
```

### Шаг 4: Готово! 🎉

Теперь для всех обновлений используйте:

```bash
cd /var/www/floodilka
sudo bash deployment/update-zero-downtime.sh
```

**Или создайте алиас для удобства:**

```bash
# Добавить в ~/.bashrc на сервере:
echo "alias upd='cd /var/www/floodilka && sudo bash deployment/update-zero-downtime.sh'" >> ~/.bashrc
source ~/.bashrc

# Теперь просто:
upd
```

## 🎁 Бонусы

### 1. Автоматический откат при ошибках

Если что-то пойдет не так, скрипт автоматически откатится к предыдущей версии:

```bash
❌ Произошла ошибка!
🔄 Откат backend...
🔄 Откат frontend...
✅ Откат завершен
```

### 2. Обновляйте когда удобно

Раньше нужно было обновлять ночью когда никого нет.
**Теперь можно обновлять когда угодно!** Пользователи ничего не заметят.

### 3. Бэкапы

Каждое обновление создает бэкап в `/var/www/floodilka-backup-*`.
Последние 3 бэкапа сохраняются автоматически.

### 4. Подробные логи

Скрипт логирует каждый шаг. Если что-то пойдет не так, вы точно будете знать что и где.

## 🆘 Что если что-то пойдет не так?

### Backend не запускается

```bash
# Смотрим логи
sudo -u floodilka pm2 logs floodilka-backend --lines 100

# Откат к бэкапу
BACKUP_DIR=$(ls -dt /var/www/floodilka-backup-* | head -1)
cd /var/www/floodilka/backend
cp -r $BACKUP_DIR/backend/* .
sudo -u floodilka pm2 reload floodilka-backend
```

### Недостаточно памяти для 2 инстансов

```bash
# Создать swap 2GB
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### Вернуться к старому способу (если очень нужно)

```bash
sudo -u floodilka pm2 delete floodilka-backend
cd /var/www/floodilka/backend
sudo -u floodilka pm2 start server.js --name floodilka-backend

# Но лучше не надо :)
```

## 🎓 Как это работает технически

1. **PM2 Cluster Mode** - запускается 2 инстанса backend вместо 1
2. **Graceful Reload** - PM2 запускает новые инстансы, ждет их готовности, и только потом останавливает старые
3. **Healthcheck** - проверка что новые инстансы отвечают перед переключением
4. **Атомарная замена frontend** - файлы заменяются мгновенно через `mv`
5. **Автоматический rollback** - при любой ошибке откат к бэкапу
