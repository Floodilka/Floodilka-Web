# Демонстрация экрана (Screen Sharing)

## Описание

Функция демонстрации экрана позволяет пользователям, находящимся в голосовом канале, делиться своим экраном с другими участниками канала. Это похоже на функционал в Discord и Google Meet.

## Как использовать

### Начать демонстрацию экрана

1. Подключитесь к голосовому каналу
2. В панели управления голосовым каналом (снизу слева) появится кнопка с иконкой демонстрации экрана
3. Нажмите на кнопку демонстрации экрана
4. Выберите окно, вкладку или весь экран, который хотите показать
5. Нажмите "Поделиться"

### Остановить демонстрацию экрана

Есть два способа остановить демонстрацию:
1. Нажмите снова на кнопку демонстрации экрана в панели управления
2. Нажмите "Прекратить демонстрацию" в панели уведомлений браузера

### Просмотр демонстрации экрана

Когда кто-то начинает демонстрацию экрана:
- Демонстрируемый экран появится в центре экрана поверх всех окон
- Внизу будет отображаться имя пользователя, который демонстрирует экран
- Если несколько пользователей демонстрируют экран одновременно, все экраны будут отображаться в сетке

## Технические детали

### Backend

**Новые события WebSocket:**
- `screen-share:start` - уведомление о начале демонстрации экрана
- `screen-share:stop` - уведомление об остановке демонстрации экрана
- `screen-share:offer` - WebRTC offer для установки соединения
- `screen-share:answer` - WebRTC answer для установки соединения
- `screen-share:ice-candidate` - ICE candidate для установки соединения

**Файлы:**
- `backend/constants/events.js` - константы событий
- `backend/websocket/voiceHandler.js` - обработчики событий screen sharing

### Frontend

**Компоненты:**
- `frontend/src/components/ScreenShare.js` - основной компонент screen sharing
- `frontend/src/components/ScreenShare.css` - стили для screen sharing
- `frontend/src/components/VoiceChannel.js` - интеграция с голосовым каналом
- `frontend/src/components/UserProfile.js` - кнопка управления screen sharing
- `frontend/src/context/VoiceContext.js` - состояние screen sharing

**Технологии:**
- **WebRTC** - для передачи видео-потока экрана
- **getDisplayMedia API** - для захвата экрана
- **Socket.IO** - для синхронизации между пользователями

## Известные ограничения

1. **Качество видео** зависит от:
   - Скорости интернет-соединения
   - Разрешения экрана
   - Производительности устройства

2. **Разрешения браузера**:
   - При первом использовании браузер запросит разрешение на захват экрана
   - Разрешение нужно предоставлять каждый раз при начале демонстрации

3. **Одновременная демонстрация**:
   - Один пользователь может демонстрировать только один экран одновременно
   - Несколько пользователей могут демонстрировать свои экраны одновременно

## Безопасность

- Демонстрация экрана работает только в голосовых каналах
- Пользователь полностью контролирует, что демонстрируется
- Браузер показывает уведомление о том, что экран демонстрируется
- Демонстрация автоматически останавливается при выходе из голосового канала

## Иконка

⚠️ **Важно**: Текущая иконка для screen sharing (`/public/icons/screen_share.png`) является временной (копия `photo.png`).

Рекомендуется заменить её на специализированную иконку для демонстрации экрана:
- Размер: 24x24px или 48x48px
- Формат: PNG с прозрачностью
- Стиль: должен соответствовать другим иконкам в приложении
