Каждый запрос к Floodilka API (REST и Gateway) должен содержать заголовок `Authorization`. Токенов три типа, у каждого свой префикс.

## Типы токенов

| Префикс | Что идентифицирует | Как получить |
|---------|--------------------|--------------|
| `Bot` | Бота приложения | Настройки приложения → вкладка «Бот» → Сбросить токен |
| `Bearer` | Пользователя, авторизовавшего ваше приложение через OAuth2 | Обмен authorization code через OAuth2 |
| *(без префикса)* | Пользователя, залогиненного в официальный клиент | Эндпоинт логина — **только для first-party** |

## Bot-токены

Префикс `Bot` используется, когда интеграция — это бот:

```http
GET /api/v1/users/@me HTTP/1.1
Host: floodilka.com
Authorization: Bot MTIzNDU2Nzg5MDEyMzQ1Njc4OQ.ZjBkYjE5ODI3NjQ4MGY4MTc1NzFjMjY1
```

Токен имеет форму `<application_id>.<secret>`. ID приложения несекретен, секретная часть — да. Доступ даёт только связка целиком.

<Aside type="tip">
	Токен можно сбросить в любой момент на вкладке **Бот**. Старые токены инвалидируются немедленно — все работающие процессы, использующие старый токен, начнут получать `401 Unauthorized`.
</Aside>

## Bearer-токены

Префикс `Bearer` используется, когда вы действуете от имени пользователя, авторизовавшего ваше приложение через OAuth2:

```http
Authorization: Bearer <access_token>
```

Bearer-токены ограничены скоупами — разрешённые операции зависят от тех скоупов, которые пользователь одобрил при авторизации. Подробнее — в [OAuth2](/topics/oauth2/).

## Отсутствующий или некорректный токен

- `401 Unauthorized` — токена нет, он некорректный или неправильный префикс
- `403 Forbidden` — токен валиден, но действие не разрешено (нет скоупа, нет прав или бот не в гильдии)

Полный список ошибок — в [Кодах ошибок](/topics/error-codes/).
