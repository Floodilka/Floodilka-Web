**Вебхук** — URL, на который любой внешний сервис может POST'ить, и результат появляется как сообщение в канале. В отличие от бота, вебхук не нужен онлайн, не требует OAuth2, не занимает место в списке участников — просто URL.

Типовые применения:
- CI/CD уведомления (GitHub Actions, GitLab, Jenkins)
- Алерты из мониторинга (Grafana, Prometheus)
- Простые интеграции, для которых бот — overkill

## Объект Webhook

```json
{
	"id": "1465...",
	"type": 1,
	"guild_id": "1424...",
	"channel_id": "1465...",
	"name": "GitHub Actions",
	"avatar": null,
	"token": "XXXXXXXXXXXXX",
	"application_id": null,
	"user": {
		"id": "974...",
		"username": "puncher"
	}
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | snowflake | ID вебхука |
| `type` | integer | `1` = Incoming, `2` = Channel Follower, `3` = Application |
| `name` | string | Отображаемое имя (override'ится в каждом запросе) |
| `avatar` | string \| null | Hash дефолтного аватара |
| `token` | string | Секрет. Любой с ним может постить — лечите как пароль |
| `channel_id` | snowflake | В какой канал постит |
| `user` | User | Кто создал |

**URL для POST'а:** `https://floodilka.com/api/v1/webhooks/:webhook_id/:webhook_token`.

## Создание

### POST /channels/:channel_id/webhooks

Требует `MANAGE_WEBHOOKS`.

```bash
curl -X POST https://floodilka.com/api/v1/channels/$CID/webhooks \
  -H "Authorization: Bot $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "GitHub Actions"}'
```

В ответе — полный объект с `token`. Сохраните URL `https://floodilka.com/api/v1/webhooks/<id>/<token>` — по нему будете постить.

## Отправка через вебхук

### POST /webhooks/:webhook_id/:webhook_token

**Авторизация не нужна** — `token` в URL и есть ключ.

```bash
curl -X POST "https://floodilka.com/api/v1/webhooks/$WID/$WTOKEN" \
  -H "Content-Type: application/json" \
```

```bash
    "content": "Деплой прошёл",
    "username": "CI",

```
	</TabItem>
	<TabItem label="С эмбедом">
```bash
curl -X POST "https://floodilka.com/api/v1/webhooks/$WID/$WTOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "embeds": [{
      "title": "PR #42 merged",
      "description": "Fix auth bug",
      "url": "https://github.com/floodilka/floodilka-web/pull/42",
      "color": 5763719
    }]
  }'
```
	</TabItem>

Query `?wait=true` сделает ответ синхронным и вернёт созданное сообщение. Без него — 204 No Content (fire-and-forget).

## Эндпоинты управления

### GET /channels/:channel_id/webhooks
### GET /guilds/:guild_id/webhooks

Список вебхуков в канале / гильдии. Требует `MANAGE_WEBHOOKS`.

### GET /webhooks/:webhook_id
### GET /webhooks/:webhook_id/:webhook_token

Получить вебхук. Второй вариант — по token'у без авторизации.

### PATCH /webhooks/:webhook_id
### PATCH /webhooks/:webhook_id/:webhook_token

Изменить (имя, аватар, канал).

### DELETE /webhooks/:webhook_id
### DELETE /webhooks/:webhook_id/:webhook_token

Удалить. Второй вариант — через token, не требует `MANAGE_WEBHOOKS` (если вам доверили токен — удалять вы тоже можете).

## Безопасность

- **Не коммитьте URL'ы вебхуков** — это фактически токен на постинг
- Если URL утёк — перегенерируйте token через `PATCH` с новым `name`, или удалите и создайте заново
- Вебхуки хорошо подходят для однонаправленных уведомлений, но не могут **читать** чат — для этого нужен полноценный бот
