# Система прав для управления сообщениями

## Описание

Добавлена новая система прав для управления сообщениями на сервере. Теперь администраторы и модераторы с соответствующими правами могут удалять чужие сообщения.

## Изменения

### Backend

1. **Новое право `manageMessages`** в модели `Role`:
   - Позволяет пользователям с этим правом удалять любые сообщения на сервере
   - Автоматически назначается роли `Administrator` при создании нового сервера

2. **Проверка прав при удалении сообщений** в `chatHandler.js`:
   - Пользователь может удалить свое сообщение
   - Владелец сервера может удалить любое сообщение
   - Пользователи с правом `manageMessages` или `manageServer` могут удалить любое сообщение

### Frontend

1. **Компонент `Chat`** обновлен:
   - Добавлена загрузка прав пользователя при входе на сервер
   - Новая функция `canDeleteMessage()` проверяет права на удаление
   - Кнопка удаления отображается только для пользователей с соответствующими правами

2. **WebSocket события**:
   - При удалении сообщения теперь передается `userId` для проверки прав

## Права доступа

Удалить сообщение могут:

1. ✅ Автор сообщения (всегда)
2. ✅ Владелец сервера
3. ✅ Пользователи с ролью, имеющей право `manageMessages`
4. ✅ Пользователи с ролью, имеющей право `manageServer`

## Миграция

Для обновления существующих серверов:

```bash
cd backend
node migrations/add-manage-messages-permission.js
```

Этот скрипт добавит право `manageMessages` всем существующим ролям `Administrator`.

## API

### Получение ролей пользователя

```http
GET /api/roles/servers/:serverId/users/:userId/roles
Authorization: Bearer <token>
```

Возвращает список ролей пользователя с их правами.

## Примеры использования

### Назначение права управления сообщениями новой роли

1. Создайте новую роль через API или интерфейс
2. Включите право `manageMessages` для этой роли
3. Назначьте роль нужным пользователям

### Проверка прав пользователя

```javascript
// Frontend
const canDelete = canDeleteMessage(message);
if (canDelete) {
  // Показываем кнопку удаления
}
```

```javascript
// Backend (WebSocket)
// Проверка происходит автоматически в chatHandler.js
socket.emit('message:delete', {
  messageId: message.id,
  userId: user.id
});
```

## Безопасность

- Все проверки прав происходят на стороне сервера
- Frontend только отображает/скрывает кнопки на основе прав
- WebSocket обработчик проверяет права перед удалением сообщения
- Системные сообщения не могут быть удалены никем
