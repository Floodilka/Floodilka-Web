# 🔒 Безопасность и шифрование в Floodilka

## Голосовые звонки

### Используемые протоколы

Floodilka использует **WebRTC** - индустриальный стандарт для реалтайм коммуникаций, который включает встроенное шифрование.

#### Технические детали:

1. **DTLS (Datagram Transport Layer Security) 1.2+**
   - Шифрует сигнальные данные и обмен ключами
   - Использует сертификаты для аутентификации пиров
   - Защита от MITM атак на уровне соединения

2. **SRTP (Secure Real-time Transport Protocol)**
   - Шифрует аудио-поток в реальном времени
   - Алгоритм: AES-128-GCM
   - Защита целостности: HMAC-SHA-256
   - Уникальные ключи для каждого соединения

3. **WebSocket Secure (WSS)**
   - TLS 1.2+ для сигнального сервера
   - Шифрует обмен offer/answer/ICE candidates

### Как это работает?

```
Пользователь A                    Сервер                    Пользователь B
     |                                |                              |
     |-- WSS (зашифровано) --------->|<----- WSS (зашифровано) -----|
     |                                |                              |
     |<========== P2P соединение (DTLS-SRTP) ======================>|
     |            ↑                                        ↑         |
     |      Зашифровано                              Зашифровано    |
     |       AES-128-GCM                             AES-128-GCM     |
```

### Уровень безопасности

| Аспект | Статус | Описание |
|--------|--------|----------|
| Шифрование трафика | ✅ **Да** | AES-128-GCM |
| Защита от прослушивания | ✅ **Да** | Трафик нельзя перехватить в открытом виде |
| Защита целостности | ✅ **Да** | HMAC предотвращает подделку пакетов |
| Forward Secrecy | ✅ **Да** | Ключи уникальны для каждой сессии |
| End-to-End шифрование | ❌ **Нет** | Сервер видит метаданные (кто, когда, сколько) |
| Сквозное шифрование контента | ⚠️ **Частично** | Контент зашифрован, но не E2E |

### Сравнение с популярными сервисами

| Сервис | Технология | E2E | Уровень |
|--------|-----------|-----|---------|
| **Floodilka** | WebRTC DTLS-SRTP | Нет | Transport encryption |
| Discord | WebRTC DTLS-SRTP | Нет | Transport encryption |
| Zoom | WebRTC DTLS-SRTP | Опционально | Transport encryption (E2E платная) |
| Google Meet | WebRTC DTLS-SRTP | Нет | Transport encryption |
| Microsoft Teams | WebRTC DTLS-SRTP | Нет | Transport encryption |
| Signal | WebRTC + E2E | **Да** | End-to-end encryption |
| WhatsApp | WebRTC + E2E | **Да** | End-to-end encryption |

**Вывод:** Floodilka использует тот же уровень шифрования, что и **Discord, Zoom, Google Meet**.

## Проверка шифрования

### Способ 1: Chrome DevTools

1. Откройте DevTools (F12) во время активного звонка
2. Перейдите на вкладку **Security**
3. Вы увидите: "Secure connection" для WSS
4. В консоли выполните:

```javascript
// Проверка статуса DTLS
navigator.mediaDevices.getUserMedia({audio: true}).then(stream => {
  console.log('✅ Медиа-поток защищён');
});
```

### Способ 2: Wireshark

Если перехватить трафик с помощью Wireshark:
- Все пакеты будут зашифрованы
- Невозможно прочитать содержимое без ключей
- Видны только IP-адреса и размеры пакетов

### Способ 3: chrome://webrtc-internals

1. Откройте `chrome://webrtc-internals` в Chrome
2. Начните звонок
3. Найдите раздел **Stats**
4. Проверьте:
   - `dtlsState: connected` ✅
   - `selectedCandidatePairState: succeeded` ✅
   - Наличие `srtpCipher` (шифр SRTP) ✅

## Что видит сервер?

### Видит (метаданные):
- ✅ Кто с кем соединяется
- ✅ Когда начался и закончился звонок
- ✅ Длительность звонка
- ✅ IP-адреса участников

### НЕ видит (контент):
- ❌ Содержимое разговора
- ❌ Аудио-данные (они зашифрованы)
- ❌ Ключи шифрования

## Часто задаваемые вопросы

### Может ли администратор сервера прослушивать звонки?

**Нет.** Аудио-данные зашифрованы между клиентами с помощью ключей, которые генерируются и обмениваются напрямую через DTLS. Сервер не имеет доступа к этим ключам и не может расшифровать трафик.

### Может ли провайдер интернета прослушивать звонки?

**Нет.** Весь трафик зашифрован с помощью SRTP. Провайдер видит только зашифрованные пакеты и метаданные (IP-адреса, размеры пакетов).

### Защищены ли звонки от правительственного прослушивания?

**Частично.** Трафик зашифрован и не может быть прослушан "на лету". Однако:
- Сервер хранит метаданные (кто, когда, с кем)
- Для полной защиты нужно E2E шифрование

### Чем отличается от Signal/WhatsApp?

Signal и WhatsApp используют **сквозное шифрование (E2E)**, где даже сервер не может расшифровать данные. Floodilka использует **транспортное шифрование** (как Discord/Zoom) - данные зашифрованы в канале, но сервер технически может получить доступ к метаданным.
