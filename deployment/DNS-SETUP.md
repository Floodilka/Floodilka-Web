# 🌐 Настройка DNS для floodilka.com и floodilka.ru

## 📋 Что нужно настроить

Для работы приложения на новых доменах нужно настроить DNS записи так, чтобы они указывали на ваш сервер.

**IP адрес сервера:** `159.89.110.44`

## ✅ Правильная конфигурация DNS

### Для floodilka.com:
```
Тип    Имя    Значение           TTL
A      @      159.89.110.44      3600
A      www    159.89.110.44      3600
```

### Для floodilka.ru:
```
Тип    Имя    Значение           TTL
A      @      159.89.110.44      3600
A      www    159.89.110.44      3600
```

## 🔧 Инструкции для популярных регистраторов

### 1. **Reg.ru**

1. Войдите в личный кабинет на [reg.ru](https://www.reg.ru)
2. Перейдите в раздел "Домены"
3. Выберите домен `floodilka.com`
4. Перейдите в раздел "DNS-серверы и зона"
5. Выберите "Использовать DNS-серверы reg.ru"
6. Нажмите "Редактировать зону"
7. Удалите все существующие A-записи
8. Добавьте новые записи:
   - A-запись: `@` → `159.89.110.44`
   - A-запись: `www` → `159.89.110.44`
9. Сохраните изменения
10. Повторите для `floodilka.ru`

### 2. **Cloudflare**

1. Войдите в [Cloudflare Dashboard](https://dash.cloudflare.com)
2. Выберите домен `floodilka.com`
3. Перейдите в раздел "DNS" → "Records"
4. Удалите существующие A-записи (если есть)
5. Добавьте новые записи:
   - Type: `A`, Name: `@`, IPv4 address: `159.89.110.44`, Proxy status: `DNS only` (серая тучка)
   - Type: `A`, Name: `www`, IPv4 address: `159.89.110.44`, Proxy status: `DNS only` (серая тучка)
6. Повторите для `floodilka.ru`

⚠️ **Важно:** Убедитесь, что proxy status = "DNS only" (серая тучка), а не "Proxied" (оранжевая тучка), иначе SSL сертификаты не получатся!

### 3. **Namecheap**

1. Войдите в [Namecheap Dashboard](https://www.namecheap.com)
2. Перейдите в "Domain List"
3. Нажмите "Manage" рядом с `floodilka.com`
4. Перейдите в раздел "Advanced DNS"
5. Удалите существующие A-записи
6. Добавьте новые записи:
   - Type: `A Record`, Host: `@`, Value: `159.89.110.44`, TTL: `Automatic`
   - Type: `A Record`, Host: `www`, Value: `159.89.110.44`, TTL: `Automatic`
7. Повторите для `floodilka.ru`

### 4. **GoDaddy**

1. Войдите в [GoDaddy](https://www.godaddy.com)
2. Перейдите в "My Products" → "Domains"
3. Нажмите на `floodilka.com`
4. Нажмите "DNS" → "Manage Zones"
5. Найдите A-записи и нажмите "Edit"
6. Удалите старые записи
7. Добавьте новые:
   - Type: `A`, Name: `@`, Value: `159.89.110.44`, TTL: `1 Hour`
   - Type: `A`, Name: `www`, Value: `159.89.110.44`, TTL: `1 Hour`
8. Повторите для `floodilka.ru`

### 5. **Другие регистраторы**

Общий принцип одинаков для всех:
1. Найдите раздел управления DNS/зоной
2. Удалите старые A-записи для @ и www
3. Добавьте новые A-записи:
   - `@` → `159.89.110.44`
   - `www` → `159.89.110.44`

## 🧪 Проверка настройки DNS

После настройки DNS подождите 5-10 минут и проверьте:

```bash
# Локально
bash deployment/check-dns.sh

# Или вручную
dig floodilka.com +short
dig floodilka.ru +short
```

Оба должны возвращать **ТОЛЬКО** `159.89.110.44`

## ⏰ Время распространения DNS

- **Минимум:** 5-15 минут
- **Обычно:** 1-2 часа
- **Максимум:** 24-48 часов (редко)

Если через 1 час DNS не обновились, проверьте:
1. Правильно ли указан IP адрес
2. Сохранились ли изменения в панели регистратора
3. Используются ли правильные DNS серверы

## ❌ Частые ошибки

### 1. Несколько IP адресов
Если `dig` возвращает несколько IP:
```bash
159.89.110.44
194.58.112.174
```

**Решение:** Удалите лишние A-записи в панели управления DNS, оставьте только `159.89.110.44`

### 2. DNS не возвращает ничего
```bash
dig floodilka.ru +short
# Пусто
```

**Решение:** DNS запись не создана. Добавьте A-запись в панели управления.

### 3. DNS указывает на другой IP
```bash
194.58.112.174
```

**Решение:** Измените значение A-записи на `159.89.110.44`

### 4. Домен использует внешние DNS серверы
Если домен использует сторонние DNS серверы (не регистратора), настройте записи там.

Проверить, какие DNS серверы использует домен:
```bash
dig floodilka.com NS +short
```
